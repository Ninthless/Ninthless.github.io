学习路线图

从零开始到能独立交付大型项目，分四个阶段。

阶段一：建立基础（1-2 天）

目标：理解 Vibe Coding 的核心思维，完成第一个端到端任务。

环境准备 → 核心概念 → 规格先行 → 快速上手
章节 你会获得什么
环境准备 搭好 AI IDE + 基础工具链
核心概念 统一术语，建立思维模型
Vibe Coding 是什么 理解与传统开发的本质差异
规格先行 学会在写代码前固定问题边界
快速上手 完成第一个完整的 Vibe Coding 任务

阶段验收：能独立写出一份可执行规格，并用它完成一个小功能。

阶段二：掌握工作流（3-5 天）

目标：建立可重复的交付流程，不再依赖运气。

任务简报 → 上下文管理 → 验证闭环 → 提示词库
章节 你会获得什么
任务简报模板 组织完整编码请求的标准格式
提示词设计原则 理解为什么有些提示词有效，有些无效
上下文管理 精准提供上下文，避免 AI 跑偏
验证闭环 每次任务后的系统性检查流程
常用提示词库 直接可用的提示词模板集合
复盘沉淀 把经验转化为可复用资产

阶段验收：连续完成 5 个任务，每次都经过完整的验证闭环，并记录复盘。

阶段三：领域深入（按需，1-2 周）

目标：在你的主要工作领域建立高效的 Vibe Coding 模式。

根据你的方向选择对应章节：

方向 章节
前端开发 前端开发
后端服务 后端服务
移动端 移动端开发
数据工程 数据工程
AI 集成 AI 集成
DevOps DevOps 流程

同时建议配合工具章节：

  • MCP 扩展：让 AI 直接访问数据库、GitHub、搜索
  • 项目规则文件：让 AI 持续遵守你的项目约定
  • Skills 与自动化：把重复工作流固化成一键触发

阶段验收：在真实项目中完成一个完整功能，从规格到上线。

阶段四：进阶与规模化（持续）

目标：在大型项目和团队中稳定应用 Vibe Coding。

章节 你会获得什么
大型项目架构 10 万行以上代码库的上下文和一致性策略
团队协作规范 团队共享提示词库和规则文件的方法
高级工作流 复杂任务的拆分和并行策略
性能优化 提升 AI 响应质量和速度的技巧
常见问题排查 AI 跑偏、上下文丢失等问题的系统性解法

快速参考

已经有基础，想直接找某类内容：

  • 写不出好提示词 → 提示词设计原则
  • AI 总是跑偏 → 上下文管理 + 规格先行
  • 不知道怎么验证结果 → 验证闭环 + 验收清单
  • 想看真实案例 → 实战案例
  • 团队推广 Vibe Coding → 团队协作规范
