三个阶段

阶段一：代码补全（2021–2022）

GitHub Copilot 的出现标志着 AI 进入开发者日常工具链。核心能力是行级和函数级补全，开发者仍然主导全部逻辑，AI 只是“更聪明的 Tab 键”。

局限性：上下文窗口极小，无法理解跨文件的设计意图。

阶段二：对话编程（2022–2024）

ChatGPT、Claude 等对话模型让开发者可以用自然语言描述需求、解释错误、重构代码。上下文窗口扩展到数万 token，多文件理解成为可能。

这个阶段的典型工作流：写代码遇到问题 → 贴给 ChatGPT → 复制结果回来。

局限性：来回切换工具，上下文丢失严重，验证全靠人工。

阶段三：Vibe Coding（2025 至今）

Andrej Karpathy 在 2025 年 2 月的推文中首次系统描述了这种模式：

“There’s a new kind of coding I call ‘vibe coding’, where you fully give in to the vibes, embrace exponentials, and forget that the code even exists.”

但真正成熟的 Vibe Coding 并非“完全放手”，而是精确分工：开发者掌控问题定义和质量判断，AI 掌控实现生成和重复性工作。

这个阶段的核心变化：

  • IDE 深度集成（Cursor、Kiro、Windsurf）
  • Agent 模式：AI 可以自主调用工具、读写文件、运行命令
  • 上下文窗口达到 100K–1M token
  • 模型理解代码库整体结构的能力大幅提升

发展趋势

Vibe Coding 正在从个人技巧演变为团队工程实践。下一个重要节点是工作流标准化：把经过验证的 Vibe Coding 流程固化为团队规范，形成可传递的能力资产。
